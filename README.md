# Math for Coding

## 1. 리포지토리 소개

저는 누구나 특별한 수학적 재능이나 많은 사전 지식이 없어도 정보·컴퓨터 분야에 발을 딛을 수 있다고 생각합니다.

하지만 코딩을 공부하다 보면 다양한 수학적 개념이 들어간 예제를 만나게 됩니다.  
또한 코딩 테스트나 알고리즘 문제를 풀다 보면 수학적 사고가 필요한 상황도 자주 마주하게 됩니다.

이 리포지토리는 그러한 상황에 대비하기 위해 만들었습니다.

이곳의 목적은 수학 자체를 깊이 연구하는 것이 아닙니다.  
수학자가 되기 위한 수학이 아니라, **코딩을 위한 수학**을 함께 정리하고 탐구하는 것이 목표입니다.

즉, 수학 공식을 단순히 외우는 것이 아니라 다음과 같은 질문을 다룹니다.

```text
이 공식은 왜 필요한가?
코딩 문제에서는 어떻게 사용되는가?
어떤 패턴을 만나면 이 개념을 떠올려야 하는가?
Python 코드로는 어떻게 구현할 수 있는가?
```

이 리포지토리는 수학을 어려워하는 사람도 천천히 따라올 수 있도록, 작은 예제에서 출발해 코딩 문제에 필요한 수학적 개념을 하나씩 정리하는 공간입니다.

---

## 2. 이 리포지토리의 방향

이 리포지토리는 다음과 같은 내용을 다룬다.

```text
1. 코딩 테스트에 자주 등장하는 수학 개념
2. 컴퓨터과학에서 기본적으로 필요한 수학적 사고
3. 공식을 외우기보다 원리를 이해하는 설명
4. Python을 이용한 간단한 구현 예제
5. 문제 해결에 바로 연결되는 수학 패턴
```

## 4. 예제: 피라미드형 바둑돌 개수 구하기

## 4.1 문제 상황

바둑돌을 다음과 같이 피라미드 모양으로 쌓는다고 하자.

```text
1단계: 1개
2단계: 3개
3단계: 6개
4단계: 10개
5단계: 15개
```

모양으로 나타내면 다음과 같다.

```text
1단계

●
```

```text
2단계

●
● ●
```

```text
3단계

●
● ●
● ● ●
```

```text
4단계

●
● ●
● ● ●
● ● ● ●
```

각 단계마다 한 줄씩 추가되고, 새로 추가되는 줄에는 단계 수만큼 바둑돌이 놓인다.

즉, `n`단계의 바둑돌 개수는 다음과 같다.

```text
1 + 2 + 3 + ... + n
```

---

## 4.2 직접 더해서 구하기

예를 들어 5단계라면 다음과 같이 계산할 수 있다.

```text
1 + 2 + 3 + 4 + 5 = 15
```

따라서 5단계의 바둑돌 개수는 15개이다.

하지만 단계가 커지면 하나씩 더하는 방식은 불편하다.

예를 들어 100단계라면 다음과 같이 모두 더해야 한다.

```text
1 + 2 + 3 + 4 + ... + 100
```

이럴 때 공식을 사용하면 빠르게 구할 수 있다.

---

## 4.3 공식

피라미드형 바둑돌의 개수는 다음 공식으로 구할 수 있다.

```text
n * (n + 1) / 2
```

Python 코드로 쓰면 다음과 같다.

```python
answer = n * (n + 1) // 2
```

여기서 `//`는 정수 나눗셈이다.

---

## 4.4 공식의 원리

공식이 갑자기 나온 것처럼 보일 수 있지만, 원리는 단순하다.

5단계를 예로 들어 보자.

```text
●
● ●
● ● ●
● ● ● ●
● ● ● ● ●
```

이 모양의 개수는 다음과 같다.

```text
1 + 2 + 3 + 4 + 5
```

이제 같은 모양을 하나 더 만들어서 거꾸로 뒤집어 붙인다고 생각한다.

```text
원래 모양       뒤집은 모양

●              ● ● ● ● ●
● ●            ● ● ● ●
● ● ●          ● ● ●
● ● ● ●        ● ●
● ● ● ● ●      ●
```

각 줄을 합치면 모두 6개가 된다.

```text
1 + 5 = 6
2 + 4 = 6
3 + 3 = 6
4 + 2 = 6
5 + 1 = 6
```

6개짜리 줄이 5줄 있으므로 전체는 다음과 같다.

```text
5 * 6 = 30
```

하지만 이것은 같은 삼각형을 2개 붙인 개수이다.

따라서 원래 삼각형 하나의 개수는 절반이다.

```text
30 / 2 = 15
```

그래서 5단계는 15개가 된다.

이를 일반화하면 다음과 같다.

```text
n단계
→ n줄이 있음

각 줄을 뒤집은 삼각형과 짝지으면
→ 한 줄마다 n + 1개가 됨

그런 줄이 n줄 있으므로
→ n * (n + 1)

하지만 삼각형 2개를 합친 것이므로
→ n * (n + 1) / 2
```

따라서 공식은 다음과 같다.

```text
1 + 2 + 3 + ... + n = n * (n + 1) / 2
```

---

## 4.5 Python 구현

```python
n = int(input("단계를 입력하세요: "))

answer = n * (n + 1) // 2

print(answer)
```

입력:

```text
5
```

출력:

```text
15
```

---

## 4.6 함수로 만들기

```python
def count_stones(n):
    return n * (n + 1) // 2


print(count_stones(1))  # 1
print(count_stones(2))  # 3
print(count_stones(3))  # 6
print(count_stones(4))  # 10
print(count_stones(5))  # 15
```

실행 결과:

```text
1
3
6
10
15
```

---

## 4.7 반복문으로도 구할 수 있다

공식을 모른다면 반복문으로도 구할 수 있다.

```python
n = int(input("단계를 입력하세요: "))

total = 0

for i in range(1, n + 1):
    total += i

print(total)
```

입력:

```text
5
```

출력:

```text
15
```

이 코드는 다음 계산을 직접 수행한다.

```text
1 + 2 + 3 + 4 + 5
```

---

## 4.8 공식과 반복문의 차이

| 방식 | 코드 | 특징 |
|---|---|---|
| 반복문 | `for i in range(1, n + 1)` | 원리를 이해하기 쉽지만 n이 커지면 반복이 많아짐 |
| 공식 | `n * (n + 1) // 2` | 한 번의 계산으로 바로 구할 수 있음 |

예를 들어 `n = 100`이라면 반복문은 100번 더하지만, 공식은 한 번에 계산한다.

```python
n = 100
answer = n * (n + 1) // 2
print(answer)
```

출력:

```text
5050
```

---

## 4.9 코딩 테스트에서의 의미

이런 형태의 문제는 코딩 테스트에서 자주 등장한다.

예를 들어 다음과 같은 문제들이 있다.

```text
1단계에 1개, 2단계에 3개, 3단계에 6개처럼 증가할 때 n단계의 개수 구하기
계단식으로 쌓인 블록 개수 구하기
삼각형 형태로 놓인 점의 개수 구하기
1부터 n까지의 합을 빠르게 구하기
```

이런 문제를 만나면 다음 패턴을 떠올리면 된다.

```text
1 + 2 + 3 + ... + n
```

그리고 이 패턴은 다음 공식으로 바꿀 수 있다.

```text
n * (n + 1) / 2
```

Python에서는 정수 결과가 필요하므로 보통 다음처럼 쓴다.

```python
n * (n + 1) // 2
```

## 5. 핵심 정리

피라미드형 바둑돌 문제는 대표적인 삼각수 예제이다.

```text
1단계 → 1개
2단계 → 1 + 2 = 3개
3단계 → 1 + 2 + 3 = 6개
4단계 → 1 + 2 + 3 + 4 = 10개
```

n단계의 전체 개수는 다음과 같다.

```text
1 + 2 + 3 + ... + n
```

이를 공식으로 나타내면 다음과 같다.

```text
n * (n + 1) / 2
```

Python 코드로는 다음처럼 작성한다.

```python
answer = n * (n + 1) // 2
```

이 공식은 단순 암기용 공식이 아니라, 같은 삼각형을 하나 더 뒤집어 붙이면 직사각형이 된다는 원리에서 나온다.

```text
삼각형 2개
→ n줄짜리 직사각형
→ 한 줄의 개수는 n + 1
→ 전체는 n * (n + 1)
→ 원래 삼각형 하나는 절반
→ n * (n + 1) / 2
```

즉, 이 예제는 코딩을 위한 수학에서 중요한 태도를 보여 준다.

```text
공식을 외우는 것보다
공식이 왜 나오는지 이해하고
그 구조를 코드로 옮기는 것이 중요하다.
```
